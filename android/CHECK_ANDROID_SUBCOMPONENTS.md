# Проверка подкомпонентов Android

## Шаг 1: Разверните "Qt 6.7.2 → Android"

1. Найдите **"Qt 6.7.2"** в списке
2. Разверните его (нажмите на стрелку вниз)
3. Найдите **"Android"** (он уже отмечен галочкой)
4. **Разверните "Android"** (нажмите на стрелку вниз рядом с "Android")
5. Внутри должны быть подкомпоненты:
   - Android armv7
   - Android arm64-v8a
   - Android x86
   - Android x86_64
   - **Android Tools** ← Может быть здесь!

## Шаг 2: Используйте поиск

1. В поле поиска (с иконкой лупы) введите: **"androiddeployqt"**
2. Или: **"Android Tools"**
3. Или: **"Tools"**

## Шаг 3: Если Android Tools нет в списке

Возможно, в Qt 6.7.2 Android Tools не доступны как отдельный компонент. В этом случае:

### Альтернативное решение: Использовать androiddeployqt из mingw_64

Хотя `androiddeployqt` из `mingw_64` не работал ранее, можно попробовать другой подход - создать минимальный Gradle проект вручную или использовать CMake напрямую.

### Или проверить другую версию Qt

Android Tools могут быть доступны только в более новых версиях Qt (например, Qt 6.7.3 или Qt 6.8+).

## Что делать сейчас:

1. **Разверните "Qt 6.7.2 → Android"** и проверьте подкомпоненты
2. **Используйте поиск** для "Android Tools" или "androiddeployqt"
3. Если не найдете, сообщите - предложу альтернативное решение
