# Решение: Android Tools недоступны в Qt 6.7.2

## Проблема

В Qt 6.7.2 Android Tools не доступны как отдельный компонент. Компонент "Android" содержит только предкомпилированные библиотеки Qt, но не `androiddeployqt`.

## Решение: Использовать Qt Creator для создания APK

В Qt 6.7.2 Qt Creator может создавать APK автоматически без отдельного `androiddeployqt`, используя встроенные механизмы.

### Шаг 1: Убедитесь, что проект собран

1. В Qt Creator выберите Kit: **Android Qt 6.7.2 Clang arm64-v8a**
2. Нажмите **Build → Build Project** (Ctrl+B)
3. Дождитесь успешной сборки

### Шаг 2: Используйте Qt Creator для создания APK

1. В Qt Creator перейдите в **Projects** (Ctrl+5)
2. Выберите Kit: **Android Qt 6.7.2 Clang arm64-v8a**
3. Перейдите на вкладку **Build**
4. В разделе **Build Steps** найдите **"Build Android APK"** или **"Deploy"**
5. Нажмите **Build** или **Run** (F5)

Qt Creator должен автоматически создать APK используя встроенные механизмы.

### Шаг 3: Если автоматическое создание не работает

Попробуйте использовать CMake напрямую для создания Gradle проекта:

1. Убедитесь, что проект собран
2. Проверьте, создался ли Gradle проект в `android-build`
3. Если да, используйте Gradle напрямую для сборки APK

## Альтернатива: Обновить до Qt 6.7.3 или новее

В более новых версиях Qt (6.7.3+) Android Tools могут быть доступны. Но это потребует пересборки проекта.

## Текущее решение

Попробуйте использовать Qt Creator для создания APK - он должен использовать встроенные механизмы без отдельного `androiddeployqt`.
